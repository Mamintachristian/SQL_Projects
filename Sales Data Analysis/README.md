# 🛒 Sample Sales Analytics with DuckDB (SQL Project)

## 📌 Project Overview
This project is a SQL-based data analysis case study using a sample sales dataset sourced from Kaggle. The objective is to practice **intermediate to advanced SQL querying techniques** using a real-world structured sales database.

The dataset was imported into a local **DuckDB database**, and multiple analytical business questions were solved using SQL.

---

## 📊 Dataset
- Source: Kaggle Sample Sales Dataset  
- Link: https://www.kaggle.com/datasets/razutowhid/sample-sales-dataset  
- Contents: Sales transactions, customers, products, orders, and related business data  
- Format: CSV files loaded into DuckDB

---

## 🛠️ Tools & Technologies
- DuckDB (analytical SQL engine)
- SQL (intermediate to advanced queries)
- Python (Jupyter Notebook environment)
- Pandas (for optional data handling)
- VS Code / JupyterLab

---

## 🧠 Project Workflow
1. Downloaded dataset from Kaggle  
2. Imported multiple CSV files into a DuckDB database  
3. Created relational structure between tables  
4. Performed SQL-based data exploration  
5. Answered 10 intermediate to hard SQL business questions generated using ChatGPT  
6. Validated outputs directly within Jupyter Notebook  

---

## 📌 Key SQL Concepts Used
- INNER / LEFT JOINs (multi-table analysis)
- Common Table Expressions (CTEs)
- Window Functions (ROW_NUMBER, RANK, SUM OVER)
- Aggregations (GROUP BY, HAVING)
- Data cleaning and transformation in SQL
- Subqueries and nested logic
- Handling duplicate and multi-region customer issues

---

## ❓ Example Business Questions Solved
- Who are the top customers based on total revenue?
- Which products generate the highest sales?
- What are the monthly sales trends?
- Which regions or countries perform best?
- How do repeat customers contribute to revenue?
- What are the top-performing product categories?
