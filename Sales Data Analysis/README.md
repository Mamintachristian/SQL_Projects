🛒 Sample Sales Analytics with DuckDB (SQL Project)
📌 Project Overview

This project is a SQL-based data analysis case study using a sample sales dataset sourced from Kaggle. The goal of this project is to practice intermediate to advanced SQL querying techniques using a real-world style sales database.

The dataset was imported into a local DuckDB database, and a series of analytical business questions were solved using SQL.

📊 Dataset
Source: Kaggle Dataset - Sample Sales Dataset
Contents: Sales transactions, customers, products, and related business entities
Format: CSV files imported into DuckDB
🛠️ Tools & Technologies
DuckDB – Lightweight analytical SQL engine
SQL (Intermediate–Advanced level queries)
Python (Jupyter Notebook) – for database connection and execution
Pandas (optional use for data handling)
VS Code / JupyterLab
🧠 Project Workflow
Downloaded dataset from Kaggle
Imported multiple CSV files into a single DuckDB database
Created relational structure between tables
Performed SQL-based data exploration
Answered 10 intermediate-to-hard analytical questions generated with ChatGPT
Validated outputs using SQL queries inside Jupyter Notebook
📌 Key SQL Concepts Practiced
JOINs (INNER, LEFT, and multi-table joins)
Window Functions (ROW_NUMBER, RANK, SUM OVER)
Common Table Expressions (CTEs)
Aggregations and group-level analysis
Handling duplicate customers and multi-region transactions
Data cleaning inside SQL (type casting, string manipulation)
Subqueries and nested logic
